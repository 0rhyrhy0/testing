깃허브에 업로드하는 과정은 스테이징 -> 커미팅 -> 업로드 로 나뉜다.

1 - 파일의 수정(생성, 삭제 포함) 이후 아무것도 하지 않은 상태
스테이징 - 커밋하기 위한 전 단계로, 임시로 저장해놓은 파일의 상태
커밋 - 깃허브에 업로드하기 직전 단계. 파일마다 커멘트를 달아놓은 상태이다.
업로드 - 업로드를 하면 실제로 깃허브 홈페이지에 수정사항이 기록되어 나중에 이용할 수 있다.

첫 생성 시
  git init - 해당 폴더를 레포지토리로 만든다. (이때 .git 파일이 생성되므로, 이후 해당 디렉토리에서만 작업해야 한다.)
  git status - 현재 스테이터스를 표시.

  파일 업로드 시
    git remote add origin "레포 주소" - 해당 폴더를 깃허브의 레포와 연동한다.
      연동 상태 확인은 git remove -v 로 가능하다.
    git remote set-url origin "새로운 주소" - 주소를 잘못 연결했을 때 다시 연결하는 방법
    
  git add . - 모든 파일을 스테이징 한다
  git commit -m "커밋내용" - 모든 파일을 커밋한다.
  
  
푸쉬
  git push


파일 변경 시 추가로 할 작업
  git status - 무엇이 변경되었는지 확인한다. 빨간색, 혹은 초록색으로 뜨는 내용이 깃허브에 등록되지 않는 변경사항들이다.
  git add .
  git commit -m "..."
  git push
  - 해당 작업은 update.bat으로 자동화 해놓았다.
